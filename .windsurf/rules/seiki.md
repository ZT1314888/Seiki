# 项目概述
## 目标
Seiki 是一个用于测量和分析户外广告的自助式 B2B 平台。每个客户都在**自己独立的数据环境中运行**：他们上传/管理自己的库存，创建活动和计划，并授权使用 Seiki 专有的移动性和人口洞察力来**衡量和预测**效果。

## 与众不同之处
1. 每个组织的数据完全独立。
2. 跨 MO/MA/BA（媒体所有者/媒体代理/品牌广告主）的自助入职服务
3. 由移动和人口数据驱动的专有 KPI 引擎。
4. 灵活的库存管理（BA/MA 可管理多个 MO 副本）。
5. 跨用户的统一分析方法。
6. 语言支持：英语。

## 技术栈
### 前端
- **框架**：React.js
- **映射与可视化**：Mapbox GL JS + deck.gl
- **用户界面结构**：模块化 React 组件，响应式容器布局（仅限桌面）

### 后台/基础设施
- **云提供商**：谷歌云平台（GCP）
- **存储**：用于资产和元数据持久化的 GCP Buckets 或 Firestore
- **数据驻留**：初始部署托管在欧盟地区；系统必须支持区域化数据存储，以确保合规性
- **全球访问**：网络应用必须可全球访问，并有可能扩展到多个地区
- **合规性**：设计支持符合 GDPR 的数据处理和存储
- **导出引擎**：服务器端生成 PDF 和 CSV 文件

# 术语和定义
## 组织类型与隔离
- **媒体所有者（MO）**：拥有户外资产（广告牌、屏幕等）。
- **媒体代理公司 (MA)**：协调多个所有者库存的广告投放。
- **品牌广告商 (BA)**：为自己的产品/服务开展广告活动；与 MA 一样，可以管理多个 MO 库存。
- **组织**：任何注册实体（MO、MA、BA）。**每个组织的数据都是严格隔离的。**

## 组织内的角色
- **管理员**：创建/修改/删除所有内容并邀请用户。
- **操作员**：只读访问权限（库存、营销活动、媒体计划、KPI）。

## 库存结构和术语
- **媒体所有者（第 1 层）**→**网络（第 2 层）**→**面孔/广告牌（第 3 层）**（唯一`面孔 ID`）。
- **无网络**：未分配广告牌的默认逻辑组（系统生成）。

## 广告系列和计划术语
- **广告系列**：具有日期和选定库存的单个广告飞行；可属于媒体计划。
- **媒体计划**：将相关广告系列（1...N）分组的容器。
- **草案广告系列**：已保存；组织内部人员可见；激活前状态。
- **激活活动**：已激活；对所有组织用户可见；运行至结束日期。
- **KPI**：由 Seiki API 提供的关键性能指标。
- **影响**：基于 24 小时移动数据的广告牌效果的可视化表达。
